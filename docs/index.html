What is A11Y?

A11ccessibility

A11Y is a numeronym for the term "accessibility". The first and last letters remain, and the number of letters removed replaces them.

You may be familiar with similar web-related numeronyms such as I18N (internationalization) and W3C (World Wide Web Consortium).

Neat. Let's dive in.

Considering Different Modes

When considering the accessibility of your project, it may be useful to consider a few different scenarios.

  • Navigation via Assistive Technology, such as a screen reader
    • Navigation via Keyboard alone

Markup

One of the simplest ways to improve the accessibilty of your site is to use the correct, semantic HTML5 elements.

Users of assistive technology will be able to navigate your site more easily, because these elements come pre-baked with some important metadata about what kind of content they contain.

Focus

Your project should be navigable by keyboard alone. If there are fancy mouse interactions, such as swiping left or right, consider adding support for keyboard users by enabling the same results with the left and right arrow keys.

Roving focus in widgets.

tabindex="0" will allow the element to be tabbed into focus within the normal document flow.

tabindex="-1" will allow the element to be programmatically focused.

One should never use a tabindex value greater than 0, because it will cause focus to unexpectedly jump around the webpage.

Images

Providing alt text on <img/> elements is very easy to do, but often overlooked.

alt text is there to assist any users that are unable to see the image, whether they are using assistive technology or are simply saving data by not loading images.
